### Simplification du format de date
**Demande :** Les dates doivent être automatiquement synchronisées avec le calendrier à la création.
**Contexte :** La fonction `fromInputDate` convertissait les dates ISO `YYYY-MM-DD` → `DD Mon YYYY` avant l'envoi au backend, ce qui rendait le flux opaque et difficile à déboguer.
**Fichier :** `next-step/src/modules/jobs/components/JobsPreviewDialog.vue`
- Suppression de `fromInputDate` — les dates restent en ISO `YYYY-MM-DD` tout au long du flux
- Correction TypeScript : `match[1]`/`match[2]` possiblement `undefined` dans `toInputDate`
- Correction TypeScript : comparaison invalide `raw === ''` sur `company_id` (type `number | null`)

**Flux des dates après correction :**
```
input type="date" → draft: YYYY-MM-DD → backend: YYYY-MM-DD → DB: date → return: YYYY-MM-DD
→ syncJobCalendarEvents(date: YYYY-MM-DD) → calendar_events créé → calendarStore.loadEvents()
```

## Applications — Correction bug reset du formulaire à la désélection d'un job

**Problème :** Désélectionner un job existant dans le formulaire réinitialisait tout le draft.

**Cause :** `watch(() => props.row)` dans `ApplicationsFormDialog` se déclenchait pendant l'édition (les deux changements `selectedRow` + `isFormOpen` sont batchés par Vue, donc le watcher voit `props.open = true`).

**Correctif :**
- Suppression du `watch(() => props.row)` séparé
- `watch(() => props.open)` initialise maintenant le draft dans les deux directions (ouverture ET fermeture)
- `watch(selectedJobId)` — ajout d'un else pour vider `draft.value.jobId` (et les autres champs) quand le job est désélectionné
